endpoints timbre.

 

El servicio de timbre fiscal tiene distintos endpoints que puedes utilizar dependiendo del tipo de procesamiento que necesitas:

Timbrado de un comprobante, donde obtienes solo el timbre
http://staging.diverza.com/stamp

Timbrado de un comprobante, donde obtienes todo el comprobante incluyendo su timbre
http://staging.diverza.com/stamp/complete

Emisión de un comprobante sin sello, donde obtienes todo el comprobante *disponible para clientes de conector fiscal
http://staging.diverza.com/issue

Cancelación de un comprobante timbrado desde nuestros servicios
http://staging.diverza.com/stamp/cancel

stamp

Al mandar tu petición de timbrado a ésta dirección, como mensaje de respuesta obtendrás solamente el complemento del timbre, tal y como se muestra a continuación.

Recuerda que todos los complementos se encuentran dentro del nodo <cfdi:Comprobante>

servicio de pruebas

  Servidor de prueba: http://staging.diverza.com/stamp
  Token de prueba: ABCD1234
  Metodo HTTP: POST
  Header HTTP: x-auth-token

ejemplo de la instrucción desde terminal

cURL http://staging.diverza.com/stamp -X POST --data @cfdi.xml -H "x-auth-token: ABCD1234" -i

issue

Ésta dirección es utilizada para el ambiente de pruebas del servicio de Conector Fiscal®, en donde los parámetros de entrada son los mismos que en stamp, con la diferencia que deberás enviar un CFDi que no haya sido sellado.

Anexo a continuación, se encuentra un ejemplo de CFDi sin el sello del comprobante, es importante incluir el atributo de certificado y noCertificado ya que son necesarios para que el sistema de Conector Fiscal lo reconozca y pueda firmar el comprobante con los certificados que hayas registrado con nosotros. También es importante incluir el atributo de "sello", pero se deberá de agregar de esta forma: sello="", es decir, sin valor, sólo el atributo, de otra manera recibiras un mensaje de error: "No válido según esquema".

La respuesta exitosa de este servicio envía todo el comprobante con el complemento de timbre.

servicio de pruebas

  Servidor de prueba: http://staging.diverza.com/issue
  Token de prueba: ABCD1234
  Metodo HTTP: POST
  Header HTTP: x-auth-token

ejemplo de la instrucción desde terminal

cURL http://staging.diverza.com/issue -X POST --data @cfdi.xml -H "x-auth-token: ABCD1234" -i
